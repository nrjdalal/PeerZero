Most problems fall into one of a few buckets. Start here.

## Slow or stalling downloads, and the wifi acting up

The classic pattern - **fast for a few seconds, then it crawls or stalls, other
downloads slow down, and the wifi gets flaky or drops** - is almost always your
**network being overwhelmed**, not PeerZero itself and (despite how it looks) usually
not your ISP.

### Too many connections are overwhelming your router (most common)

BitTorrent opens a lot of simultaneous connections. A consumer router has a limited
NAT/connection table and CPU, and a torrent client can flood it - at which point
**everything** on the machine degrades: DNS lookups crawl, unrelated downloads stall,
and the wifi drops. The torrents suffer too, because they can't resolve trackers or
reach peers. It's easy to blame the ISP, but if killing the download engine instantly
restores your normal speed (test it with a plain download), it's connection overload,
not throttling.

**What helps:** lower the per-torrent connection limit. PeerZero already defaults to a
conservative **25**; if your router is still struggling, drop it further via the
`TORRENT_MAX_CONNS` environment variable before starting the engine:

```bash
TORRENT_MAX_CONNS=15 bun run dev
```

Fewer connections is gentler on the router and usually _more_ stable - and often
faster overall, because the rest of your network keeps working.

### Your ISP is throttling BitTorrent (less common)

Some ISPs use deep packet inspection to detect and throttle BitTorrent. If lowering
the connection limit doesn't help and the same slowdown happens with every torrent
client, this may be it. **Use a VPN** - it hides the traffic so there's nothing to
single out. Connect the VPN first, then start downloads.

### It's genuinely a low-seed torrent

If one torrent is slow but others are fine, it may just not have many seeders. Check
the **Seeds** count - see the next section.

## A torrent is stuck at "Fetching" or 0%

The torrent has **no seeders**, so there's nobody to download from. This is common
with old or obscure releases. Remove it and try a better-seeded torrent - the seed
count is the single best predictor of a successful download.

## Nothing downloads at all

- **Check the engine indicator.** The toolbar shows **Engine online** when the
  download sidecar is running. If it says offline, the engine process isn't up -
  restart the dev stack (`bun run dev`).
- **VPN/firewall blocking peers.** If a VPN or firewall blocks the torrent ports, no
  peers connect. Try a different VPN server or check the firewall.
